# Meeting Intelligence Skill

**Notion 기반 회의 준비** - Notion에서 정보를 수집하고, Claude의 리서치로 보강하여 회의 자료를 자동으로 생성합니다.

---

## 🎯 이 스킬이 뭔가요?

회의를 완벽하게 준비하기 위해 **Notion에서 맥락을 수집**하고, **Claude의 지식으로 보강**한 후, **전문적인 회의 자료**를 자동으로 생성합니다.

내부 팀을 위한 상세한 사전 읽기 자료와 모든 참석자를 위한 외부 의제를 함께 만들어 회의를 구조화하고 모두를 같은 페이지에 맞춥니다.

---

## 🔄 어떻게 작동하나요?

### 7단계 프로세스

```text
1. 회의 맥락 이해 → 2. Notion에서 검색 → 3. 내용 분석 →
4. Claude로 보강 → 5. 내부 사전 읽기 생성 → 6. 외부 의제 생성 → 7. 문서 연결
```

#### 1단계: 회의 맥락 이해

**목표**: 회의의 목적과 참석자 파악

```
수집할 정보:
- 회의 주제/제목
- 참석자 (내부 팀 + 외부 참석자)
- 회의 목적 (의사결정, 브레인스토밍, 상태 업데이트, 고객 데모 등)
- 회의 유형 (내부 전용 vs 외부 참석자 포함)
- 관련 프로젝트/이니셔티브
- 다룰 구체적인 주제들
```

#### 2단계: Notion에서 검색

**도구**: `Notion:notion-search`

```
찾아야 할 항목:
- 회의 주제와 관련된 프로젝트 페이지
- 이전 회의 노트
- 사양서 또는 설계 문서
- 관련 작업 또는 이슈
- 최근 업데이트 및 보고서
- 고객/파트너 정보 (해당하는 경우)

검색 전략:
- 주제 기반: "모바일 앱 리디자인"
- 프로젝트 범위: 프로젝트 teamspace 내 검색
- 작성자별 필터: created_by_user_ids 활용
- 날짜 범위: created_date_range 필터로 최근 업데이트만
```

#### 3단계: 내용 분석

**도구**: `Notion:notion-fetch`

```
각 관련 페이지에서 추출할 정보:
- 프로젝트 상태 및 타임라인
- 최근 의사결정 및 업데이트
- 미해결 질문 또는 막힘 요소
- 관련 메트릭 또는 데이터
- 이전 회의의 액션 아이템

정보 간격 파악: 뭔가 빠진 부분이 있나요?
```

#### 4단계: Claude 리서치로 보강

**목표**: Notion 정보 이상의 가치 추가

```
기술 회의의 경우:
- 복잡한 개념을 폭넓은 청중을 위해 설명
- 업계 표준 관행 요약
- 경쟁 상황 분석
- 평가 기준 제안

고객 회의의 경우:
- 회사 배경 조사 (공개 정보)
- 논의 주제와 관련된 업계 트렌드
- 해당 섹터의 일반적인 문제점
- 유사 고객의 모범 사례

의사결정 회의의 경우:
- 의사결정 프레임워크 (예: RICE)
- 위험 평가 패턴
- 트레이드오프 분석 방식
- 성공 기준 제안

⚠️ 주의: 일반적 지식만 사용. 구체적인 사실을 지어내지 마세요.
```

#### 5단계: 내부 사전 읽기 생성

**도구**: `Notion:notion-create-pages`

```
제목: "[회의 주제] - 사전 읽기 (내부)"

문서 구조:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📅 회의 개요
  - 날짜, 시간, 참석자, 목적

📚 배경 맥락
  - 이 회의가 뭔가요? (2-3문장)
  - 왜 중요한가요? (비즈니스 맥락)
  - 관련 Notion 페이지 링크

📊 현재 상태
  - 현재 어디에 있나요? (Notion 내용)
  - 최근 업데이트 및 진행 상황
  - 주요 메트릭 또는 데이터

💡 맥락 & 인사이트 (Claude 리서치)
  - 업계 맥락 또는 모범 사례
  - 관련 고려사항
  - 논의할 수 있는 가능한 접근법

🎯 핵심 논의 사항
  - 공중파 시간이 필요한 주제들
  - 풀어야 할 미해결 질문
  - 필요한 의사결정

📌 이 회의에서 필요한 것
  - 예상 결과
  - 해야 할 의사결정
  - 정의할 다음 단계
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

대상: 내부 참석자만
목적: 회의 전에 팀에 완전한 맥락과 정렬 제공
```

#### 6단계: 외부 의제 생성

**도구**: `Notion:notion-create-pages`

```
제목: "[회의 주제] - 의제"

문서 구조:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 회의 상세 정보
  - 날짜, 시간, 참석자

🎯 목표
  - 명확한 회의 목표 (1-2문장)

⏱️ 의제 항목 (시간 배분 포함)
  1. 주제 1 (10분)
  2. 주제 2 (20분)
  3. 주제 3 (15분)

💬 논의 주제
  - 다룰 핵심 항목들
  - 답변해야 할 질문들

✅ 필요한 의사결정
  - 명확한 의사결정 포인트들

📝 액션 아이템
  - (회의 중 작성될 예정)

🔗 관련 리소스
  - 관련 페이지 링크
  - 사전 읽기 문서 링크
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

대상: 모든 참석자 (내부 + 외부)
목적: 회의 구조화, 시간 관리, 명확한 초점 유지
톤: 전문적이고, 명확하고, 깔끔함
```

#### 7단계: 문서 연결

```
1️⃣ 사전 읽기를 의제에 연결:
   - 의제에서: "배경은 <mention-page>사전 읽기</mention-page>를 참고하세요"

2️⃣ 두 문서를 프로젝트에 연결:
   - 프로젝트 페이지 업데이트
   - "회의" 섹션에 추가

3️⃣ 교차 참조:
   - 의제는 사전 읽기를 내부 참석자에게 언급
   - 사전 읽기는 의제의 회의 구조 언급
```

## 📊 회의 유형별 구조

### 의사결정 회의

```
회의 상세 정보 → 목표 → 옵션 (장단점) → 권장사항 → 논의 → 의사결정 → 액션 아이템
```

### 상태 업데이트

```
회의 상세 정보 → 프로젝트 상태 → 진행도 → 예정된 작업 → 막힘 요소 → 논의 → 액션 아이템
```

### 고객/외부 회의

```
회의 상세 정보 → 목표 → 시간별 의제 → 논의 주제 → 다음 단계
```

### 브레인스토밍

```
회의 상세 정보 → 목표 → 제약조건 → 아이디어 → 논의 → 다음 단계
```

## 💡 Claude 리서치 보강 패턴

Notion 내용만으로는 부족합니다. Claude의 지식으로 추가 가치를 제공하세요.

| 맥락 | 추가할 인사이트 |
| --- | --- |
| **기술 맥락** | 기술 설명, 업계 표준, 솔루션 비교, 평가 기준 |
| **비즈니스 맥락** | 업계 트렌드, 경쟁 상황, 일반적인 도전 과제, ROI 고려사항 |
| **의사결정 지원** | 의사결정 프레임워크, 위험 평가, 트레이드오프 분석, 성공 기준 |
| **고객 맥락** | 업계별 도전 과제, 일반적인 문제점, 유사 회사의 모범 사례, 가치 제안 |
| **프로세스 지도** | 회의 진행 기법, 논의 프레임워크, 회고 패턴, 브레인스토밍 구조 |

⚠️ **핵심**: 일반적인 지식과 분석 능력만 사용. Notion 사실과 Claude 인사이트 구분 필요.

## 🚀 시작하기

### 사용자 입장

```
"[회의 주제]에 대한 회의를 준비해줄래?
참석자는 [이름들], 날짜는 [날짜]야."
```

**예시:**
```
"우리 모바일 앱 리디자인 회의를 준비해줄래?
월요일 2시에 디자인팀과 엔지니어링팀이 참석해.
우리는 새로운 네비게이션 구조를 최종 결정해야 해."
```

### 기대할 수 있는 것

1. **상세한 사전 읽기**: Notion 정보 + Claude 인사이트, 내부용
2. **전문적인 의제**: 시간 배분, 의사결정 포인트, 액션 아이템 자리 포함
3. **완벽하게 준비됨**: 모든 참석자가 회의에 앞서 준비할 수 있음
4. **상호 연결됨**: Notion과 문서가 양방향으로 연결됨
5. **즉시 공유 가능**: 전문적이고 폴리싱된 결과

## 📚 스킬의 핵심 철학

> **"완벽한 회의는 완벽한 준비에서 시작된다"**

- Notion은 **과거의 사실**: 프로젝트 상태, 결정, 진행도
- Claude는 **미래의 가치**: 맥락, 인사이트, 프레임워크
- 함께하면: **모든 사람이 같은 페이지에서 시작**
- 결과: **효율적이고 생산적인 회의**

## ✅ 모범 사례

1. **두 문서 모두 만들기**: 중요한 회의는 사전 읽기 + 의제
2. **출처 표시하기**: Notion vs Claude 인사이트 구분
3. **넓게 시작하기**: Notion 검색을 광범위하게 한 후 좁혀나가기
4. **간결하게**: 사전 읽기는 최대 2-3페이지
5. **외부용 다듬기**: 의제는 광택 있고 초점 있게
6. **신중하게 보강하기**: Claude 리서치는 가치를 더해야 함 (펑크가 아님)
7. **문서 연결**: 사전 읽기 ↔ 의제 상호 언급
8. **메트릭 포함**: Notion의 데이터로 논의 근거 마련
9. **적절하게 공유**: 사전 읽기는 내부팀에만, 의제는 모든 참석자에게
10. **미리 공유**: 중요한 회의는 24시간 이상 미리 공유

## ❓ FAQ

**Q: Notion에서 관련 정보를 못 찾으면?**

A: 검색 범위를 넓혀보세요. 다양한 검색 용어 시도, 사용자에게 페이지 URL 요청, 날짜 범위 조정.

**Q: 내부 회의에는 사전 읽기가 꼭 필요한가요?**

A: 중요한 회의는 필수. 간단한 상태 업데이트는 의제만 가능하지만, 의사결정 회의는 반드시 둘 다.

**Q: 외부 고객 회의는?**

A: 의제는 항상 만들기. 사전 읽기는 내부팀만 만들거나 매우 간단하게.

**Q: Claude 리서치를 너무 많이 추가하면 안 되나요?**

A: 피하세요. Notion 사실에 기반한 구체적 인사이트만 추가. 일반적 강연처럼 보이면 안 됨.

**Q: 회의 중에 문서를 업데이트하나요?**

A: 네, 의제를 업데이트하세요:
- 의사결정: 각 결정과 근거 + 책임자 기록
- 액션 아이템: 체크박스 리스트 + 담당자 + 마감일
- 주요 결과: 주요 결과들 정리

**Q: 같은 회의가 반복되면?**

A: 회의 시리즈 부모 페이지 만들기: 일정, 회의노트 리스트, 정해진 의제, 액션 아이템 추적기. 개별 회의를 여기에 연결.
