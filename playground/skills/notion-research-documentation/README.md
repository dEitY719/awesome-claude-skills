# Research & Documentation Skill

**Notion 기반 연구 및 문서화** - 산재된 정보를 검색하고, 종합하여 체계적인 연구 자료로 변환합니다.

---

## 🎯 이 스킬이 뭔가요?

Notion 워크스페이스 전체에서 정보를 **광범위하게 검색**하고, 관련 페이지에서 **상세 내용을 수집**한 후, 이를 **하나의 체계적인 문서**로 종합합니다.

산재된 정보들을 정확한 출처 인용과 함께 구조화된 보고서로 변환하여, 의사결정에 필요한 종합적인 맥락을 제공합니다.

---

## 🔄 어떻게 작동하나요?

### 4단계 프로세스

```text
1. 검색 → 2. 페칭 → 3. 합성 → 4. 문서 생성
```

#### 1단계: 정보 검색

**도구**: `Notion:notion-search`

```
연구 주제로 검색:
- Notion 워크스페이스 전체에서 관련 페이지 찾기
- 특정 teamspace으로 범위 제한 (필요한 경우)
- 검색 결과에서 가장 관련 높은 페이지 파악

검색 전략:
- 광범위한 키워드로 시작: "시장 동향", "AI 기술"
- 점진적 좁혀나가기: 결과 검토 후 필터 추가
- 다양한 용어로 재검색: 놓친 정보 확인
```

#### 2단계: 페이지 내용 수집

**도구**: `Notion:notion-fetch`

```
각 관련 페이지에서 추출:
- 핵심 정보 및 주요 내용
- 구체적인 데이터, 수치, 인용구
- 최종 편집 날짜 (최신성 확인)
- 페이지 링크 (나중에 인용 시 사용)

기록할 사항:
✓ 출처 페이지 URL
✓ 주요 발견 사항
✓ 통계 및 데이터
✓ 배경 정보
```

#### 3단계: 정보 종합

```
수집한 정보 분석:

📌 핵심 주제 파악
  - 반복되는 주제와 패턴 식별
  - 여러 출처에서 공통된 의견 찾기

🔗 개념 연결
  - 관련된 개념들 간의 연결고리
  - 인과관계 파악
  - 논리적 흐름 구성

⚠️ 정보 검증
  - 출처 간 일치 여부 확인
  - 상충되는 정보 표시
  - 정보 격차 파악

📊 논리적 조직
  - 타임라인, 카테고리, 중요도별 정렬
  - 명확한 계층 구조 수립
```

#### 4단계: 문서 생성

**도구**: `Notion:notion-create-pages`

```
3가지 출력 형식 중 선택:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 연구 요약 (2-3페이지)
   간결하고 명확한 발견 요약
   바쁜 의사결정자용

📊 포괄적 보고서 (5-10페이지)
   상세한 배경, 분석, 인용
   깊이 있는 이해 필요 시

⚡ 빠른 브리프 (1-2페이지)
   핵심만 압축한 요약
   빠른 개요 필요 시
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

모든 문서에 포함:
✓ 명확한 제목과 실행 요약
✓ 논리적 섹션 구분
✓ 출처로 돌아가는 링크 (mentions)
✓ 실행 가능한 결론 또는 다음 단계
```

## 📊 출력 형식 선택

| 형식 | 대상 | 길이 | 사용 시기 |
| --- | --- | --- | --- |
| **연구 요약** | 의사결정자 | 2-3페이지 | 빠른 결정 필요 |
| **포괄적 보고서** | 깊이 있는 분석 | 5-10페이지 | 전략 수립, 투자 결정 |
| **빠른 브리프** | 매니저, 임원 | 1-2페이지 | 정말 빠른 개요 |

**자동 선택**: 사용자가 명시하지 않으면 **연구 요약**이 기본값

## 💡 실제 사용 예시

### 예시 1: 시장 조사

```
사용자 요청:
"우리 경쟁사들의 가격 전략을 Notion에서 조사해줄 수 있어?
포괄적인 보고서로 정리해 줘."

프로세스:
1️⃣ 검색: "경쟁사 가격", "가격 분석", "경쟁 분석"
2️⃣ 페칭: 각 경쟁사 페이지에서 가격 정보 수집
3️⃣ 합성: 가격 비교표, 트렌드, 우리의 위치 파악
4️⃣ 문서: 포괄적 보고서 + 권장사항

결과:
└── 경쟁사 가격 전략 분석 (Notion 페이지)
    ├── 실행 요약
    ├── 경쟁사별 가격 비교
    ├── 시장 트렌드
    ├── 우리의 위치 분석
    ├── 권장 전략
    └── 출처 링크
```

### 예시 2: 기술 조사

```
사용자 요청:
"우리 기술 스택 문서에서 마이크로서비스 아키텍처에 대해
모든 정보를 찾아서 빠른 브리프로 정리해줄래?"

프로세스:
1️⃣ 검색: "마이크로서비스", "아키텍처", "기술 스택"
2️⃣ 페칭: 설계 문서, 결정 기록, 구현 가이드
3️⃣ 합성: 왜, 어떻게, 현황 파악
4️⃣ 문서: 빠른 브리프 + 다음 단계

결과:
└── 마이크로서비스 아키텍처 개요
    ├── 핵심 개념
    ├── 우리의 구현 상태
    ├── 주요 이점
    ├── 도전 과제
    └── 추천 참고 자료
```

### 예시 3: 경쟁사 분석

```
사용자 요청:
"3개 스타트업에 대한 모든 조사 자료를 찾아서
비교 분석 보고서를 만들어 줄래?"

프로세스:
1️⃣ 검색: 각 스타트업명, "경쟁분석", "시장조사"
2️⃣ 페칭: 각사 개요, 기술, 가격, 투자 정보
3️⃣ 합성: 병렬 비교, 강점/약점, 시장 기회
4️⃣ 문서: 포괄적 보고서

결과:
└── 경쟁사 비교 분석
    ├── 개요 테이블
    ├── 회사별 상세 분석
    ├── 직접 비교
    ├── 시장 포지셔닝
    ├── 기회 및 위협
    └── 전략 권장사항
```

## 🚀 시작하기

### 사용자 입장

```
"[주제]에 대해 Notion에서 조사해줄 수 있어?
[형식]으로 정리해 줘."
```

**형식 옵션:**
- "빠른 브리프로" → 1-2페이지 요약
- "포괄적 보고서로" → 상세 분석 (기본값: 연구 요약)
- (명시하지 않으면) → 연구 요약 (2-3페이지)

**예시:**
```
"시장의 AI 자동화 도구 트렌드를 조사해줄래?
포괄적 보고서로 정리해 줘.
우리가 어디에 포지셔닝해야 할지까지 분석해 줄래?"
```

### 기대할 수 있는 것

1. **체계적인 검색**: 광범위 → 집중으로 단계적 진행
2. **정확한 출처**: 모든 주장에 링크된 출처
3. **명확한 구조**: 읽기 쉬운 섹션과 포맷팅
4. **종합적 분석**: 여러 출처의 정보를 하나로 통합
5. **실행 가능한 인사이트**: 데이터 기반 결론
6. **Notion 저장**: 팀 전체가 접근 가능한 문서

## 📚 스킬의 핵심 철학

> **"정보는 흩어져 있지만, 인사이트는 종합에서 나온다"**

- Notion은 **정보의 보고**: 프로젝트, 결정, 분석이 흩어져 있음
- 검색과 수집은 **정보 모으기**: 관련된 모든 내용 수집
- 종합은 **의미 찾기**: 패턴, 연결, 격차 파악
- 문서화는 **공유 가능한 형태**: 팀 전체가 이해할 수 있도록

**결과**: 산재된 정보 → 체계적 분석 → 실행 가능한 인사이트

## ✅ 모범 사례

1. **광범위하게 시작**: 처음엔 넓은 검색 → 점진적 좁혀나가기
2. **항상 인용하기**: 모든 정보에 출처 링크 필수
3. **날짜 확인**: 페이지 수정 날짜로 최신성 검증
4. **크로스 검증**: 여러 출처에서 정보 확인
5. **명확하게 구조화**: 헤딩, bullet, 포맷팅으로 가독성 ↑
6. **차별화 명시**: 사실 vs 분석 vs 권장사항 구분
7. **초점 유지**: 요청한 주제에만 집중
8. **격차 표시**: 찾지 못한 정보나 상충 표시
9. **다음 단계 제시**: 추가 조사 또는 실행 계획 포함
10. **팀과 공유**: Notion 링크로 팀 전체 접근 가능하게

## ❓ FAQ

**Q: 검색 결과가 없으면 어떻게 하나요?**

A: 다양한 전략 시도:
- 더 광범위한 키워드 사용
- 다른 용어로 재검색
- 특정 teamspace으로 범위 확대/축소
- 사용자에게 관련 페이지 URL 요청

**Q: 검색 결과가 너무 많으면?**

A: 범위 좁혀나가기:
- 더 구체적인 키워드로 필터
- 특정 페이지 내에서만 검색
- 날짜 범위 제한
- 사용자에게 범위 명확히

**Q: 여러 출처에서 정보가 모순되면?**

A: 명확하게 표시하기:
- 각 관점을 다른 섹션에서 제시
- 어느 출처가 더 최신인지 표시
- 권장 해석 제시 (단, 근거 포함)
- 사용자에게 알림

**Q: 페이지 접근 권한이 없으면?**

A: 사용자에게 문의:
- 어떤 페이지에 접근 불가인지 명시
- 권한 확인 요청
- 대신 다른 방식 제시 (수동 요약 등)

**Q: 출처 인용은 어떻게?**

A: Notion mentions 사용:
- 각 정보 옆에 출처 페이지 mention
- 연결된 참고 자료 섹션
- 참고문헌 목록

**Q: 보고서를 어디에 저장하나요?**

A: 세 가지 옵션:
- **기본**: 최상위 독립 페이지로 생성
- **지정된 부모 페이지**: `page_id` 매개변수 사용
- **특정 database**: database 테이블에 생성

**Q: 정보가 너무 구식이면?**

A: 명확하게 표시:
- 마지막 검증 날짜 표시
- 정보의 나이 언급
- 업데이트 필요성 제시
- 더 최신 정보 검색 제안

**Q: 기술 용어가 복잡하면?**

A: 쉽게 설명하기:
- 핵심 개념 미리 정의
- 비유나 예시 포함
- 배경 정보 섹션 추가
- 전문 용어는 각주로 설명
